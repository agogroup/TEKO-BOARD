# テスト実行

プロジェクトの品質チェックを実行します。

## 実行内容

### 1. TypeScript 型チェック

ビルドを実行して型エラーを検出します。

```bash
npm run build
```

### 2. ESLint

コードスタイルと潜在的な問題をチェックします。

```bash
npm run lint
```

### 3. ユニットテスト（存在する場合）

```bash
npm run test 2>/dev/null || echo "テストスクリプトが未設定です"
```

## 出力形式

各チェックの結果を以下の形式で報告してください:

```
## 型チェック
状態: ✅ 成功 / ❌ 失敗

エラー内容（失敗時）:
- ファイル:行番号 - エラーメッセージ

## ESLint
状態: ✅ 成功 / ⚠️ 警告あり / ❌ エラー

問題箇所:
- ファイル:行番号 - ルール名 - メッセージ

## テスト
状態: ✅ 成功 / ❌ 失敗 / ⏭️ スキップ

失敗テスト:
- テスト名 - 理由
```

## 修正提案

エラーや警告がある場合は、修正方法を具体的に提案してください。
